# User Story Map

## 1. Onboarding (The Resurrection)
- **Epic:** Data Ingestion
  - *Story:* As a user, I can upload a batch of invoice photos so that I don't have to type them manually.
  - *Story:* As a user, I can upload my WhatsApp chat exports so Otto understands my supplier communications.
- **Epic:** State Generation
  - *Story:* As a user, I want to see a live build of my digital inventory so I know the system is extracting data correctly.
  - *Story:* As a user, I can confirm the final digital state with one click to finalize setup.

## 2. Daily Operations (Approval Feed)
- **Epic:** Task Management
  - *Story:* As a user, I want to see pending actions (like reorders) in a clear feed so I can review my daily tasks.
  - *Story:* As a user, I can approve or reject a suggested action so that I maintain control over business decisions.
- **Epic:** Domain Playbooks
  - *Story:* As a user, I can select a playbook (e.g., HR, Healthcare) to populate my feed with industry-specific tasks.

## 3. Automation & Trust (The Autonomy Ladder)
- **Epic:** Earning Autonomy
  - *Story:* As a user, I want Otto to ask for permission to automate a task only after I have approved it multiple times.
  - *Story:* As a user, I can grant autonomous execution rights to specific action types.
- **Epic:** Safety & Revocation
  - *Story:* As a user, I can view a Trust Meter showing what Otto is allowed to do.
  - *Story:* As a user, I can click "Undo" on an automated action within 1 hour if a mistake was made.
  - *Story:* As a user, I can toggle off autonomy for any action instantly.
